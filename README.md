# XM Browser — نسخه اولیه (MVP)

این پوشه یک پروژه‌ی کامل Android Studio (به زبان Kotlin) است که شامل:
- مرورگر مبتنی بر WebView
- مدیریت چند تب هم‌زمان
- بوک‌مارک (ذخیره، نمایش، حذف)
- حالت تاریک / روشن (Dark Mode)
- صفحه خانه اختصاصی با برند XM
- آیکون اولیه برنامه (X قرمز در پس‌زمینه + M طلایی)

## مراحل راه‌اندازی (برای کسی که تازه با Android Studio کار می‌کند)

### ۱. نصب Android Studio
از سایت رسمی دانلود کنید: developer.android.com/studio
نسخه‌ی Windows / Mac / Linux را متناسب با سیستم‌عامل خود نصب کنید. در مرحله نصب، گزینه‌های پیش‌فرض (Standard) را قبول کنید تا SDK اندروید هم خودکار نصب شود.

### ۲. باز کردن پروژه
1. این پوشه (XMBrowser) را از حالت zip خارج کنید (Extract).
2. در Android Studio گزینه‌ی **Open** را بزنید و پوشه‌ی XMBrowser را انتخاب کنید.
3. صبر کنید تا «Gradle Sync» تمام شود (برای اولین بار به اینترنت نیاز دارد چون وابستگی‌ها و خود Gradle دانلود می‌شوند — معمولاً چند دقیقه طول می‌کشد).

### ۳. اجرای برنامه
- **روی شبیه‌ساز (Emulator):** از منوی بالا Device Manager را باز کنید، یک دستگاه مجازی (مثلاً Pixel) بسازید، سپس دکمه‌ی ▶ Run را بزنید.
- **روی گوشی واقعی:** در گوشی خود «حالت توسعه‌دهنده» و «USB Debugging» را فعال کنید، گوشی را با کابل وصل کنید، و دکمه‌ی ▶ Run را بزنید.

اگر در حین Sync با خطا مواجه شدید، معمولاً کافیست File → Sync Project with Gradle Files را دوباره بزنید.

## ساختار پروژه
```
app/src/main/java/com/xm/browser/   کدهای Kotlin (MainActivity, مدیریت تب، بوک‌مارک)
app/src/main/res/layout/            چیدمان صفحات
app/src/main/res/values/            رنگ، متن، تم روشن
app/src/main/res/values-night/      تم تاریک
app/src/main/res/drawable/          آیکون‌های وکتور
app/src/main/res/mipmap-*/          آیکون برنامه
app/src/main/assets/home.html       صفحه خانه
```

## نکته درباره آیکون
آیکون فعلی یک نسخه‌ی برنامه‌ای ساده (تولیدشده با کد) از ایده‌ی شماست. برای نسخه‌ی نهایی و حرفه‌ای پیشنهاد می‌شود یک طراح آن را در Figma/Illustrator بسازد، یا از ابزار داخلی Android Studio به نام **Image Asset Studio** (راست‌کلیک روی res → New → Image Asset) برای جایگزینی استفاده کنید.

## یک محدودیت فنی مهم
پشتیبانی از افزونه‌های Chrome (Chrome Extension) در یک مرورگر اندرویدی از نظر فنی ممکن نیست — حتی خود Chrome برای اندروید هم این قابلیت را ندارد، چون به معماری دسکتاپ کروم وابسته است. به‌جای آن می‌توان یک سیستم Userscript (مثل Tampermonkey) برای تزریق اسکریپت سفارشی در صفحات اضافه کرد که در فاز بعدی قابل پیاده‌سازی است.

## مراحل بعدی پیشنهادی (طبق اولویت‌بندی رایج چنین پروژه‌هایی)
1. Ad Block داخلی (مسدودسازی دامنه‌های تبلیغاتی با لیست فیلتر)
2. حالت ناشناس (Incognito) با WebView مجزا و بدون ذخیره‌سازی
3. تاریخچه‌ی بازدید
4. دانلود منیجر (تشخیص لینک فایل/ویدیو + دکمه دانلود + مدیریت دانلودها)
5. تنظیمات پیشرفته: User Agent، حالت دسکتاپ، موتور جستجو، فونت، زوم
6. Reader Mode، QR Scanner، اشتراک‌گذاری سریع، اسکرین‌شات

هر کدام از این موارد را که بخواهید، می‌توانیم در گفتگوی بعدی روی همین پروژه اضافه کنیم.
